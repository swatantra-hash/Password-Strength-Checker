About PassGuard

The thinking behind the tool.

The Problem

Most password strength meters use simplistic rules — "has uppercase? has a number?" — that give users a false sense of security. A password like P@ssword1 passes most checks despite being trivially crackable. Users deserve better.

The Solution

PassGuard uses information-theoretic entropy, pattern recognition (keyboard walks, common passwords, repetition), and real breach data from Have I Been Pwned to provide a holistic, accurate security score.

Technical Architecture

Under the hood.

🧮

Entropy Engine

Calculates bits of entropy using log₂(pool_size ^ length). The pool is dynamically determined from the character classes present in the password, giving an accurate measure of randomness.

🔍

Pattern Detector

Scans for keyboard sequences (qwerty, asdf), character repetition (aaa), sequential runs (abc, 123), and checks against a list of the top 200 most common passwords.

🛡️

k-Anonymity Breach Check

Your password is SHA-1 hashed client-side. Only the first 5 characters of the hash are sent to the HIBP API. The full hash and password never leave your browser, preserving complete privacy.

🎲

Crypto-Secure Generator

Uses crypto.getRandomValues() for true randomness instead of Math.random(). Fisher-Yates shuffle ensures uniform distribution. Guarantees at least one character from each selected type.

Scoring Formula

base_score = entropy_points (max 50)
+ criteria_points (max 30)
+ length_bonus (max 20)
penalties = common_password (-60)
+ keyboard_pattern (-15)
+ repeated_chars (-10)
+ sequential_chars (-10)
final_score = clamp(base_score - penalties, 0, 100)
